This folder contains short notes documenting successful AI coding agent behaviors worth repeating.

The purpose is to capture patterns where an agent avoided a plausible but wrong change by checking the current code, respecting project boundaries, and preserving the intended architecture.

These notes exist to support future agent usage by making good judgment calls easier to recognize and repeat.

Each note documents a real incident and typically includes:
- the original request or review feedback,
- the tempting but incorrect action,
- the verification step that changed the outcome,
- and the principle that should be reused.

What this is not:
- This is not a praise log.
- This is not a substitute for code review.
- This is not a reason to ignore feedback without verification.
